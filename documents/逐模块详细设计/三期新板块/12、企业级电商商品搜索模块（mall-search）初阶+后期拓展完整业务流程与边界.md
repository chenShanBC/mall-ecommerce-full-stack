# 企业级电商商品搜索模块（mall\-search）初阶+后期拓展完整业务流程与边界

本文档整合商品搜索模块 MVP最小可行版本 与 后期拓展完整版业务，明确各阶段业务流程、职责边界，贴合mallFei微服务架构，兼顾当前落地与后期迭代，可直接用于项目开发、文档归档、技术评审。

# 一、搜索模块定位（结合微服务拆分·后期完整版）

搜索模块（mall\-search）是电商平台的**商品分发、用户找货、流量转化**核心模块，独立承载全平台商品搜索、筛选、排序、联想推荐、搜索风控、搜索数据分析全流程，属于**高并发、高可用、重体验、强关联**的核心业务域。

与商品、用户、订单、营销、库存深度协同，但**搜索索引构建、检索逻辑、排序规则、推荐策略、搜索风控**完全自治，不侵入其他微服务业务逻辑，是连接用户需求与商品供给的关键枢纽，遵循微服务高内聚、低耦合的核心拆分原则，实现搜索业务的独立部署、扩展与优化，解决海量商品场景下用户高效找货的核心需求。

结合微服务拆分（后期完整版），完整搜索模块包含五大核心域，贴合领域驱动设计（DDD）的限界上下文划分，确保各域职责单一、数据独立，同时适配电商搜索的核心场景需求：

1. 搜索索引域：商品索引构建、索引更新、索引优化、索引同步，基于全文检索引擎（如Elasticsearch）实现商品数据的高效检索，支撑海量商品快速查询。

2. 检索核心域：关键词检索、分类检索、筛选检索、搜索纠错、非法词过滤、特定跳转，覆盖用户各类搜索场景，确保检索结果精准匹配用户需求。

3. 排序推荐域：搜索排序规则、个性化推荐、热门搜索、搜索联想、相关推荐，优化搜索结果展示逻辑，提升用户找货效率与转化效率。

4. 搜索风控域：恶意搜索拦截、搜索频率限制、异常搜索监控、搜索作弊识别，保障搜索服务稳定可用，避免恶意请求影响服务性能。

5. 运营管控域：搜索规则配置、搜索数据统计、热门词管理、搜索日志分析、检索效果优化，支撑运营人员精细化运营，持续提升搜索体验。

# 二、整体架构说明

本搜索模块基于微服务拆分设计，核心遵循**索引独立、检索高效、数据不跨域、体验优先**原则，坚守微服务数据私有、接口交互的核心准则，适配高并发搜索场景，确保搜索响应速度与检索精准度，同时与其他服务解耦：

- mall\-auth：仅做鉴权，不参与搜索索引构建、检索逻辑与排序规则制定，仅提供用户身份校验支撑。

- mall\-user：提供用户ID、用户标签、搜索历史、浏览偏好、会员等级等信息，用于个性化搜索推荐与搜索历史留存，支撑个性化找货场景。

- mall\-product：提供商品基础信息、SKU信息、商品分类、品牌、属性、售价、上下架状态等数据，作为搜索索引的核心数据源，支撑检索与筛选功能。

- mall\-inventory：提供商品库存状态（现货/缺货）、库存数量（仅展示范围，不暴露具体数值），用于搜索结果筛选（如仅显示有货商品），提升搜索实用性。

- mall\-marketing：提供商品优惠信息、活动标签（如秒杀、满减），用于搜索结果排序加权与标签展示，助力营销转化。

- mall\-search：**唯一负责搜索索引构建与更新、商品检索、筛选排序、推荐联想、搜索风控、数据统计**，独享自身业务数据（索引数据、搜索日志、规则配置），不允许其他服务直接访问数据库与索引库，仅通过接口提供搜索服务，支撑全平台搜索场景。

核心边界：

- 只做“商品搜索与推荐相关”，不修改商品信息、不处理库存扣减、不参与订单创建与支付，仅提供搜索结果支撑与数据同步。

- 所有搜索操作（检索、排序、推荐、索引更新）均有日志记录、可追溯，搜索响应速度满足用户体验要求（MVP阶段响应时间≤300ms，拓展阶段≤100ms），适配高并发场景。

- 搜索索引与商品数据实时同步，确保搜索结果与商品实际状态（上下架、价格、库存）一致，避免误导用户。

- 检索逻辑与排序规则可配置、可优化，支持运营人员根据业务需求调整，适配不同阶段的搜索运营策略，同时兼顾用户体验与商业目标。

# 三、初阶阶段业务流程（当前落地核心）

核心目标：支撑商城基础商品搜索场景，完成关键词检索、基础筛选、简单排序、搜索历史留存，无复杂个性化推荐与精细化风控，满足用户基础找货需求，实现搜索与商品、库存的基础协同，确保检索精准与响应高效。

## 3\.1 核心业务流程（7大必备）

1. **搜索索引初始化与同步（后台自动）**：系统初始化时，同步mall\-product模块的商品基础数据（商品ID、名称、分类、品牌、基础售价、上下架状态）、mall\-inventory模块的库存状态，构建初始搜索索引；商品信息（如上下架、价格）更新时，mall\-product、mall\-inventory模块主动回调mall\-search，触发索引实时更新，确保索引与商品实际状态一致，支撑精准检索，这是搜索功能正常运行的基础保障。

2. **关键词检索（C端）**：用户登录经mall\-auth鉴权通过后，在搜索框输入商品关键词（如“手机”“连衣裙”），提交搜索请求；系统对关键词进行基础清洗（去除空格、特殊字符），调用检索接口查询搜索索引，匹配包含关键词的商品，返回基础搜索结果（商品ID、名称、主图、售价、库存状态），满足用户基础找货需求，这是搜索模块的核心入口功能。

3. **基础筛选（C端）**：搜索结果页支持基础筛选功能，用户可按商品分类、品牌、价格区间（如0\-100元、100\-200元）、库存状态（仅显示有货）筛选结果；系统根据用户筛选条件，对检索结果进行二次过滤，返回符合条件的商品，帮助用户快速缩小找货范围，提升找货效率。

4. **基础排序（后台配置\+自动）**：搜索结果默认按“综合排序”展示（结合商品销量、上架时间、匹配度排序），运营人员可在后台配置排序权重（如优先展示销量高的商品）；支持用户手动切换排序方式（销量从高到低、价格从低到高/从高到低、上架时间从新到旧），系统根据排序规则重新整理搜索结果，适配不同用户的找货偏好。

5. **搜索历史（C端）**：系统自动记录用户的搜索关键词（去重），用户再次进入搜索框时，可查看历史搜索记录，点击即可快速重新搜索；支持用户手动删除单个/全部搜索历史，保护用户隐私，同时提升用户重复找货的便捷性，贴合用户使用习惯。

6. **搜索日志与基础统计（运营后台）**：系统记录所有搜索请求（关键词、搜索时间、用户ID、检索结果数量、点击情况）；运营人员可在后台查看核心统计数据（搜索量、热门关键词、检索无结果关键词、搜索转化率），支持按时间筛选，为运营优化提供基础数据支撑，助力后续搜索体验提升。

7. **搜索异常处理（后台自动\+手动）**：当搜索请求超时、索引查询失败时，系统自动返回友好提示（如“搜索繁忙，请稍后再试”），同时记录异常日志；运营人员可在后台查看异常记录，手动触发索引重建、重启搜索服务，确保搜索服务稳定可用，减少用户体验损耗。

# 四、后期拓展完整版业务流程（企业级升级）

在MVP基础上，补齐个性化推荐、精细化检索、搜索风控、智能优化、多场景适配，形成企业级搜索中台，遵循微服务独立扩展原则，提升搜索精准度、响应速度与转化效率，适配高并发、多场景、精细化运营需求，同时融入电商搜索的核心优化逻辑，贴合用户找货习惯与商业转化目标。

## 4\.1 新增拓展业务流程（10大核心，覆盖五大核心域）

1. **搜索索引精细化管理（搜索索引域）**：优化索引结构，新增商品属性索引（如服装的尺码、颜色，家电的型号、配置）、商品详情索引，提升检索精准度；支持索引分片、索引预热，适配海量商品（千万级以上）检索场景，降低检索延迟；新增索引备份与恢复机制，避免索引丢失，同时支持索引灰度更新，不影响线上搜索服务，确保服务稳定性。

2. **精细化检索功能（检索核心域）**：新增关键词纠错功能，用户输入错别字、拼音（如“阿迪斯”纠正为“阿迪达斯”）时，自动匹配正确关键词并返回检索结果，降低用户检索无结果的概率；新增非法词过滤，维护非法词库，屏蔽涉黄、涉赌等敏感关键词，不返回相关检索结果，同时减轻服务器压力，保障内容合规；新增特定跳转功能，配置关键词与店铺、活动页的映射关系，用户搜索特定关键词（如“阿迪达斯”）时，可直接跳转至对应店铺或活动专场，提升活动曝光与转化效率；新增多维度检索（如语音搜索、图片搜索），通过识别组件将语音、图片转为文字后进行检索，丰富用户搜索方式，适配不同用户习惯。

3. **个性化搜索推荐（排序推荐域）**：基于用户搜索历史、浏览记录、下单偏好、用户标签，实现个性化检索结果排序（如用户常浏览女装，优先展示女装相关商品）；新增搜索联想功能，用户输入关键词时，实时推荐相关关键词（如输入“手机”，推荐“手机壳”“手机充电器”），并显示对应商品数量，引导用户精准搜索；新增相关商品推荐，在搜索结果页底部展示与检索商品相关的商品，提升用户浏览时长与转化率；新增热门搜索词展示，在搜索框下方展示平台热门搜索词，引导用户搜索，提升热门商品曝光。

4. **搜索排序精细化配置（排序推荐域）**：运营人员可在后台配置多套排序规则（如日常排序、活动排序、新品排序），根据业务场景（如大促、新品上市）切换启用；支持排序权重精细化调整（如大促期间，活动商品排序权重提升）；新增商品权重体系，结合商品销量、评价、转化率、优惠力度等多维度计算商品权重，确保排序结果兼顾用户体验与商业目标，同时支持权重规则的动态调整与灰度测试。

5. **搜索风控精细化（搜索风控域）**：新增恶意搜索拦截（如频繁重复搜索、批量恶意检索），限制异常用户的搜索频率，保护搜索服务；新增搜索作弊识别（如恶意刷搜索排名），对作弊行为进行记录与拦截，确保搜索结果公平；搭建搜索风控预警机制，设置异常阈值（如某IP搜索频率超过阈值），自动推送预警信息，运营人员及时处理，同时支持风控规则的自定义配置，适配不同场景的风控需求。

6. **多场景搜索适配（检索核心域）**：新增分类搜索，用户可通过商品分类（如家电、服装、食品）直接检索对应品类商品，结合分类属性进行精准筛选；新增品牌搜索，支持用户直接搜索品牌名称，展示该品牌下所有商品，同时支持品牌筛选与排序；新增跨境商品搜索，适配跨境电商场景，支持按关税、清关状态筛选，同步跨境商品相关信息，满足跨境找货需求；新增新品搜索、秒杀商品搜索，单独划分搜索入口，提升特定商品曝光。

7. **搜索数据精细化统计与分析（运营管控域）**：搭建搜索数据看板，实时展示核心数据（搜索量、检索成功率、无结果率、搜索转化率、热门关键词、用户搜索偏好、各品类搜索占比）；支持按时间、用户等级、区域、商品品类筛选数据；生成搜索分析报告，分析用户搜索行为（如搜索无结果原因、用户偏好品类）、检索效果瓶颈，为运营优化（如商品上新、关键词优化）提供数据支撑；支持自定义报表导出（PDF/Excel格式），满足不同运营需求，助力精细化运营决策。

8. **搜索规则自动化优化（运营管控域）**：引入智能算法，根据搜索数据（如关键词点击量、转化率）自动优化检索逻辑与排序权重，减少人工操作；支持A/B测试，运营人员可对不同检索规则、排序规则进行测试，选择最优方案上线；新增关键词自动优化，根据用户搜索行为，自动推荐可优化的关键词（如无结果关键词、低转化关键词），引导运营人员调整商品标题与搜索配置，提升检索精准度。

9. **搜索日志全量管理（运营管控域）**：全量记录搜索日志（包括用户搜索行为、检索结果、点击情况、筛选操作），支持日志检索、导出，用于问题追溯、用户行为分析；新增搜索异常全量监控，记录所有异常请求（超时、失败、风控拦截），分析异常原因，优化搜索服务性能；支持日志留存，满足合规要求，同时为后续搜索优化提供数据支撑。

10. **搜索体验优化（全域联动）**：优化搜索响应速度，通过索引优化、缓存策略（如热门搜索结果缓存），将搜索响应时间控制在100ms以内，适配高并发场景（如大促峰值）；新增搜索结果标签展示（如“秒杀”“满减”“新品”），突出商品优势，提升用户点击意愿；优化筛选功能，新增多维度筛选（如商品材质、尺寸、评分），支持筛选条件组合，帮助用户快速找到目标商品；新增搜索历史同步，用户在多端（H5、小程序）登录后，同步搜索历史，提升跨端用户体验。

# 五、完整业务边界（通用）

## 5\.1 核心职责

负责五大核心域（搜索索引域、检索核心域、排序推荐域、搜索风控域、运营管控域）的规则制定、业务落地、数据管理；为商品、用户、营销、库存提供搜索索引同步、检索结果支撑、搜索数据反馈等能力；独立完成搜索索引构建与更新、商品检索、筛选排序、推荐联想、搜索风控、数据统计，不侵入任何外部微服务业务逻辑，坚守微服务高内聚、低耦合原则，实现搜索业务的独立自治，同时支撑全平台搜索场景的高效运转。

## 5\.2 对内边界

- → mall\-product：提供商品基础信息、SKU信息、分类、品牌、属性、上下架状态等数据；接收搜索模块的索引同步请求，商品信息更新时主动回调搜索模块，触发索引更新；不修改商品其他数据，仅提供数据支撑与回调通知，通过接口实现数据交互，不直接访问商品数据库。

- → mall\-user：读取用户ID、用户标签、搜索历史、浏览偏好、会员等级等信息；不修改用户任何数据，仅用于个性化推荐、搜索历史留存、用户行为分析，确保用户信息安全，同时支撑个性化搜索场景。

- → mall\-inventory：提供商品库存状态、库存范围信息；接收搜索模块的库存同步请求，库存状态更新时主动回调搜索模块，触发索引更新；不修改库存其他数据，仅提供数据支撑，用于搜索结果筛选，确保搜索结果与库存状态一致。

- → mall\-marketing：提供商品优惠信息、活动标签；接收搜索模块的优惠信息同步请求，优惠活动更新时主动回调搜索模块，触发索引更新；不修改营销其他数据，仅提供数据支撑，用于搜索结果排序加权与标签展示，助力营销转化。

- → mall\-auth：提供用户鉴权服务，校验用户身份合法性；不参与搜索业务逻辑，仅提供鉴权结果，确保搜索接口访问安全，支撑个性化搜索与搜索历史功能。

## 5\.3 对外边界

- 对外开放接口：MVP阶段开放关键词检索、基础筛选、排序、搜索历史、索引同步接口；后期拓展后，新增个性化推荐、搜索联想、语音/图片搜索、风控预警、数据统计等接口。

- 接口访问范围：仅对C端H5、小程序、PC运营后台、商家后台开放；第三方平台不开放任何核心接口，仅可调用公开商品检索接口（需鉴权，且不返回敏感数据），避免核心搜索逻辑与数据泄露。

- 数据安全要求：所有接口返回数据需脱敏（不暴露商品库存具体数值、商家敏感信息、用户完整信息）；搜索日志、索引数据、规则配置全程留存，可追溯，满足合规要求；坚守微服务数据私有原则，不泄露自身业务数据（如排序规则、索引结构），同时保护用户搜索隐私。

- 鉴权要求：除公开商品检索接口外，其余所有接口均需通过mall\-auth模块的Sa\-Token鉴权与角色校验（如运营接口需管理员角色，商家接口需商家角色，C端接口需用户登录鉴权）；搜索接口支持限流控制，避免高并发请求压垮服务，确保服务高可用。

## 5\.4 禁止范围

- 不修改商品信息、不上下架商品、不处理商品属性变更（仅同步商品数据用于索引构建）；

- 不直接操作商品库存、不管理库存业务（仅同步库存状态用于搜索筛选）；

- 不处理订单创建、支付、售后等业务（仅提供搜索结果支撑，不参与后续转化流程）；

- 不做用户登录鉴权、Token分发、用户信息修改（统一由mall\-auth认证中心、mall\-user模块承担）；

- 不跨模块修改其他领域数据（如商品价格、用户标签、营销规则），不接收其他领域的业务指令（仅提供数据查询与搜索支撑）；

- 不允许篡改搜索结果、不允许违规调整排序规则、不允许泄露搜索核心逻辑（如排序权重、索引结构）；

- 不存储用户密码、完整身份证号、未脱敏手机号等敏感用户数据（仅存储用户ID用于关联），不存储商品完整库存、商家核心信息等非搜索类敏感数据。
