# 企业级电商数据统计模块（mall\-statistics）初阶\+后期拓展完整业务流程与边界

本文档整合数据统计模块 MVP最小可行版本 与 后期拓展完整版业务，明确各阶段业务流程、职责边界，贴合mallFei微服务架构，兼顾当前落地与后期迭代，可直接用于项目开发、文档归档、技术评审。

# 一、数据统计模块定位（结合微服务拆分·后期完整版）

数据统计模块（mall\-statistics）是电商平台的**数据采集、数据处理、统计分析、报表展示、数据服务**核心模块，独立承载全平台所有业务数据（订单、商品、用户、营销、库存、文件等）的采集、清洗、转换、统计、分析与可视化展示全流程，属于**高可用、高并发、强精准、可扩展**的核心业务域。

与订单、库存、用户、营销、文件、定时任务等所有核心微服务深度协同，但**数据采集、数据处理、统计分析、报表生成、数据接口服务**完全自治，不侵入其他微服务业务逻辑，是支撑平台精细化运营、业务决策、风险管控的关键枢纽，遵循微服务高内聚、低耦合的核心拆分原则，实现数据统计业务的独立部署、扩展与优化，解决全平台数据分散、统计效率低、分析维度单一、数据不可追溯、决策缺乏数据支撑的核心需求，同时支持海量数据处理，适配高并发数据采集与统计场景，为运营、运维、管理层提供精准的数据支撑。

结合微服务拆分（后期完整版），完整数据统计模块包含五大核心域，贴合领域驱动设计（DDD）的限界上下文划分，确保各域职责单一、数据独立，同时适配电商数据统计的全场景需求，兼顾数据精准性与处理效率：

1. 数据采集域：全平台业务数据采集（实时采集、定时采集）、数据来源管理、采集规则配置、采集权限管控、采集异常处理，支持多数据源适配，确保数据采集的全面性、实时性与准确性，为后续数据处理与分析提供基础。

2. 数据处理域：数据清洗（去重、去噪、补全）、数据转换（格式标准化、字段映射）、数据脱敏、数据存储（实时存储、离线存储）、数据归档，处理采集到的原始数据，形成标准化、高质量的统计数据，支撑统计分析的精准性。

3. 统计分析域：核心指标统计（订单、商品、用户等核心指标）、多维度分析（时间、区域、用户群体、商品分类等）、趋势分析、对比分析、异常数据识别，挖掘数据背后的业务规律，为运营决策提供支撑。

4. 报表展示域：标准化报表生成、自定义报表配置、报表可视化展示、报表导出、报表权限管控，支持多终端适配，让不同角色快速获取所需数据报表，提升数据使用效率。

5. 数据服务域：数据接口提供、数据查询服务、数据订阅服务、数据权限管控、数据安全审计，为各业务模块、运营后台、第三方系统提供标准化数据服务，支撑跨模块数据协同与业务拓展。

# 二、整体架构说明

本数据统计模块基于微服务拆分设计，核心遵循**数据精准、采集实时、处理高效、安全可控**原则，坚守微服务数据私有、接口交互的核心准则，适配高并发数据采集、海量数据处理场景，确保数据统计时效与准确性，同时与其他服务解耦，采用分布式数据处理架构，支持水平扩展，提升系统可维护性与扩展性，贴合企业级数据统计服务的高可用与高精准要求：

- mall\-auth：仅做鉴权，不参与数据采集、处理、分析等逻辑，仅提供管理员、运营人员、运维人员的身份校验与角色权限支撑，确保数据统计操作（报表查看、数据查询、配置管理）的安全性与权限合规。

- mall\-user：向数据统计模块提供用户相关数据（用户注册、用户活跃、用户消费、用户等级等），接收数据统计模块的用户相关统计结果回调（如用户画像、用户留存分析），支撑用户维度数据统计与分析，不修改用户核心数据，仅提供数据支撑。

- mall\-order：向数据统计模块提供订单相关数据（订单创建、订单支付、订单取消、订单完成、订单金额等），接收数据统计模块的订单相关统计结果回调（如订单趋势、订单转化率分析），支撑订单维度数据统计与业务优化，提升订单处理效率。

- mall\-inventory：向数据统计模块提供库存相关数据（库存数量、库存变动、库存预警、库存周转率等），接收数据统计模块的库存相关统计结果回调（如库存健康度分析、库存积压预警），支撑库存精细化管理与优化。

- mall\-marketing：向数据统计模块提供营销相关数据（活动参与、优惠券领取使用、营销转化、活动效果等），接收数据统计模块的营销相关统计结果回调（如营销活动ROI分析、优惠券效果分析），支撑营销活动优化与精细化运营。

- mall\-file：向数据统计模块提供文件相关数据（文件上传量、下载量、文件类型占比、存储容量占用等），接收数据统计模块的文件相关统计结果回调（如文件使用趋势分析），支撑文件资源管理优化。

- mall\-job：向数据统计模块提供定时任务相关数据（任务执行量、执行成功率、异常率等），同时接收数据统计模块的统计任务需求，执行定时数据采集、定时报表生成等任务，支撑数据统计的自动化运转，降低人工运维成本。

- mall\-message：接收数据统计模块的消息推送请求（如异常数据告警、报表生成通知），执行消息推送操作，同步消息推送状态至数据统计模块，支撑数据统计相关通知的及时触达。

- mall\-statistics：**唯一负责数据采集、数据处理、统计分析、报表展示、数据服务**，独享自身业务数据（采集规则、处理规则、统计结果、报表配置、数据日志），不允许其他服务直接访问数据库，仅通过接口提供数据查询、报表获取、数据订阅等服务，支持分布式数据处理（如Spark、Flink），采用实时\+离线混合处理模式，支持水平扩展，支撑全平台数据统计场景，同时可适配容器化部署，降低运维成本、提升扩展性。

核心边界：

- 只做“数据统计相关”，不修改其他业务模块核心数据、不参与业务逻辑决策，仅接收各模块数据、完成数据采集处理与统计分析、提供数据服务与报表支撑，专注于数据价值挖掘核心能力，遵循微服务高内聚低耦合原则。

- 所有数据操作（采集、清洗、转换、统计、查询、导出）均有日志记录、可追溯，数据统计时效满足业务需求（MVP阶段实时数据采集延迟≤15s，离线统计延迟≤1h；拓展阶段实时数据采集延迟≤5s，离线统计延迟≤10min），适配高并发数据采集场景（如大促期间订单数据实时采集），同时通过分布式处理避免数据堆积，应对峰值数据压力。

- 数据统计统一管理，支持多数据源适配、多维度统计分析，确保数据精准性与一致性，同时支持数据脱敏与权限管控，保护敏感数据安全，贴合合规要求。

- 支持数据统计结果实时回调与报表自动化生成，异常数据及时推送告警，确保数据价值快速落地；同时支持自定义统计规则与报表配置，适配不同角色的数据需求，提升数据使用效率。

# 三、初阶阶段业务流程（当前落地核心）

核心目标：支撑商城基础数据统计场景，完成核心业务数据的基础采集、处理、统计与报表展示，覆盖订单、商品、用户、营销四大核心场景，为运营与管理层提供基础数据支撑，实现基础数据统计自动化，降低人工统计成本，采用简单高效的数据处理方式，降低初期开发成本，同时完成基础数据架构搭建，支撑核心业务数据统计落地。

## 3\.1 核心业务流程（7大必备）

1. **基础数据采集配置（运营/运维后台）**：运维人员经mall\-auth鉴权\+管理员角色校验通过后，配置基础数据采集规则，选择数据来源（mall\-order、mall\-user、mall\-marketing、mall\-inventory），设置采集频率（实时采集、每小时采集、每日采集），配置采集字段（如订单金额、用户注册时间、商品销量），校验采集规则合法性，完成基础数据采集配置，确保采集的数据贴合初期统计需求。

2. **基础数据采集（后台自动）**：系统根据配置的采集规则，自动从各业务模块采集基础业务数据，支持实时采集核心数据（如订单创建、支付数据）、定时采集非核心数据（如用户活跃、商品库存变动）；采集过程中校验数据完整性，针对采集失败的数据，自动重试1次（间隔1分钟），重试失败后标记采集异常，记录异常日志，确保数据采集的完整性与及时性。

3. **基础数据处理（后台自动）**：采集到原始数据后，系统自动进行基础数据处理，包括数据去重（删除重复数据）、数据补全（补全缺失的核心字段）、格式标准化（统一数据格式），处理完成后将标准化数据存储至数据统计数据库，更新数据处理状态（处理成功、处理失败），处理失败时标记异常，通知运维人员排查，确保数据质量符合统计要求。

4. **基础统计分析（后台自动）**：系统根据预设的核心统计指标，对处理后的标准化数据进行基础统计分析，包括订单统计（每日订单量、订单金额、支付转化率）、用户统计（每日注册量、活跃用户数、用户留存率）、商品统计（商品销量、热销商品TOP10）、营销统计（活动参与人数、优惠券使用率），统计完成后生成基础统计结果，更新统计状态，确保统计结果精准可靠。

5. **基础报表展示与导出（运营/管理层后台）**：运营人员、管理层经mall\-auth鉴权\+对应角色校验通过后，可在后台查看基础标准化报表（日报、周报、月报），报表涵盖核心统计指标，支持按时间范围（今日、昨日、近7天、近30天）筛选查看；支持报表导出（Excel格式），便于数据归档与二次分析，满足基础数据查看与使用需求。

6. **数据统计异常基础处理（后台自动\+手动）**：当数据采集失败、处理失败、统计结果异常（如统计数值明显异常）时，系统自动记录异常日志，标记异常类型（采集异常、处理异常、统计异常）；针对核心数据异常，自动推送告警消息至运维人员；运维人员可在后台查看异常记录，手动触发数据重新采集、处理与统计，排查异常原因，确保核心统计数据的准确性，同时记录异常处理过程，为后续优化提供支撑。

7. **基础数据服务与回调（后台自动）**：系统向各业务模块提供基础数据查询接口（如订单统计数据、用户统计数据），支撑各模块业务优化；统计完成后，自动回调关联的业务模块，同步核心统计结果（如商品销量统计、用户留存统计），确保各业务模块及时掌握自身业务数据情况，支撑业务流程的闭环协同，同时记录回调日志，便于问题追溯。

# 四、后期拓展完整版业务流程（企业级升级）

在MVP基础上，补齐多数据源适配、海量数据处理、精细化统计分析、自定义报表、智能预警、数据安全管控，形成企业级数据统计中台，遵循微服务独立扩展原则，提升数据采集实时性、处理效率、统计维度与数据安全性，适配高并发、海量数据、精细化运营需求，完善数据全生命周期管理，同时融入智能分析与风控机制，为平台决策提供更精准、更全面的数据支撑，降低人工数据分析成本。

## 4\.1 新增拓展业务流程（10大核心，覆盖五大核心域）

1. **多数据源精细化采集（数据采集域）**：新增多数据源适配（内部微服务、第三方平台、日志数据、埋点数据），支持自定义数据源接入，配置数据源访问权限；新增采集规则精细化配置（按字段优先级采集、条件过滤采集、增量采集），避免无效数据采集；新增采集监控，实时监控采集状态、采集速度、采集成功率，针对采集异常制定差异化处理策略（如核心数据源采集失败立即告警，非核心数据源延迟重试）；支持采集任务批量配置与管理，适配海量数据采集场景，同时支持数据采集限流，避免高并发采集压垮业务模块。

2. **海量数据高级处理（数据处理域）**：引入分布式数据处理框架（如Spark、Flink），支撑海量数据（亿级以上）实时\+离线混合处理，提升数据处理效率；新增数据脱敏精细化配置，针对不同类型敏感数据（用户手机号、订单支付信息）制定差异化脱敏规则，保护用户隐私与业务数据安全；新增数据分区存储（按时间、数据类型分区），提升数据查询与统计效率；新增数据归档精细化管理，配置归档规则（如超过6个月的离线数据自动归档），支持归档数据查询与恢复；新增数据质量校验，建立数据质量评估体系，自动识别数据缺失、异常、不一致等问题，推送优化建议，确保数据质量。

3. **精细化统计分析（统计分析域）**：新增多维度统计分析（时间维度：时/日/周/月/季度/年；空间维度：区域、门店；用户维度：用户画像、消费层级、用户标签；商品维度：商品分类、品牌、规格），挖掘数据背后的业务规律；新增趋势分析（同比、环比、预测分析），预测业务发展趋势（如销量预测、用户增长预测）；新增异常数据智能识别，通过算法识别异常数据（如异常订单、异常销量），推送告警并分析异常原因；新增自定义统计指标配置，支持运营人员根据业务需求创建个性化统计指标，适配精细化运营需求；新增跨模块数据关联分析（如订单数据与用户数据关联、库存数据与商品销量关联），提供更全面的数据分析视角。

4. **自定义报表与可视化（报表展示域）**：新增自定义报表配置，支持运营人员拖拽式创建报表，选择统计指标、筛选条件、展示样式（表格、折线图、柱状图、饼图、热力图）；新增报表可视化大屏，实时展示核心数据指标（平台整体运营数据、各业务模块核心数据），支持多终端适配（PC端、大屏端）；新增报表订阅功能，支持按时间（每日、每周、每月）自动推送报表至指定人员（邮箱、企业微信）；新增报表权限精细化管控，不同角色仅可查看对应权限范围内的报表，确保数据安全；支持报表模板保存与复用，提升报表创建效率，同时支持报表数据钻取，点击报表数据可查看明细数据，便于深度分析。

5. **数据服务精细化（数据服务域）**：新增数据接口精细化配置，支持接口权限管控、接口限流、接口调用日志记录，确保数据接口安全可靠；新增数据查询精细化，支持多条件组合查询、模糊查询、范围查询，提升数据查询效率；新增数据订阅服务，支持各业务模块订阅指定数据（如实时订单数据、用户活跃数据），数据更新时自动推送，支撑业务模块实时数据需求；新增数据API网关，统一管理数据接口，支持接口版本管理、接口文档自动生成，便于其他模块集成与调试；支持数据批量查询与导出，适配大数据量导出场景，同时支持数据格式自定义（Excel、CSV、PDF）。

6. **数据权限精细化管控（全域联动）**：新增角色权限精细化配置，区分管理员、运维人员、运营人员、管理层、商家的权限（如商家仅可查看自身店铺相关数据，运营人员可查看全平台运营数据）；支持数据行级、列级权限控制，确保不同角色仅能访问对应权限的数据；新增操作权限日志记录，全量记录所有数据操作（查询、导出、配置修改），实现操作可追溯；支持临时权限授予，便于临时数据分析与运维操作；新增数据访问审计，定期审计数据访问记录，排查数据安全风险，确保数据安全合规。

7. **智能数据预警与决策支撑（统计分析域\+报表展示域）**：新增智能预警规则配置，支持按统计指标设置预警阈值（如订单量骤降、库存积压、用户流失率过高），配置预警级别（紧急、普通、提醒）与预警渠道（企业微信、短信、邮件）；新增智能决策建议，通过算法分析统计数据，推送业务优化建议（如调整营销活动策略、优化商品库存、针对性用户运营）；新增预警处理流程，支持预警认领、处理、反馈，形成预警闭环管理；新增数据驾驶舱，整合核心数据指标、预警信息、决策建议，为管理层提供一站式决策支撑，提升决策效率与准确性。
