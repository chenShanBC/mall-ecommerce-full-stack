# 企业级电商文件服务模块（mall\-file）初阶+后期拓展完整业务流程与边界

本文档整合文件服务模块 MVP最小可行版本 与 后期拓展完整版业务，明确各阶段业务流程、职责边界，贴合mallFei微服务架构，兼顾当前落地与后期迭代，可直接用于项目开发、文档归档、技术评审。

# 一、文件服务模块定位（结合微服务拆分·后期完整版）

文件服务模块（mall\-file）是电商平台的**文件存储、上传下载、安全管控、资源管理**核心模块，独立承载全平台所有文件（商品图片、用户头像、订单凭证、营销素材、合规文件等）的上传、下载、存储、转换、查询、删除、异常处理全流程，属于**高可用、高并发、强安全、多场景**的核心业务域。

与用户、商品、订单、营销、售后、运营等所有核心微服务深度协同，但**文件存储管理、上传下载调度、文件格式转换、安全校验、资源管控**完全自治，不侵入其他微服务业务逻辑，是支撑全平台文件类资源高效流转、安全存储的关键枢纽，遵循微服务高内聚、低耦合的核心拆分原则，实现文件服务业务的独立部署、扩展与优化，解决全平台文件存储分散、上传下载效率低、安全管控不足、资源难以管理的核心需求，同时支持多存储介质灵活切换，降低后期维护与扩展成本。

结合微服务拆分（后期完整版），完整文件服务模块包含五大核心域，贴合领域驱动设计（DDD）的限界上下文划分，确保各域职责单一、数据独立，同时适配电商文件服务的全场景需求，兼顾性能与安全：

1. 文件存储域：文件存储介质管理、存储策略配置、文件持久化存储、存储容量监控、多存储介质适配（本地存储、云存储），实现文件的安全、高效持久化，支持存储介质的灵活切换与扩展，贴合企业级文件存储的高性能与高可用需求。

2. 文件上传下载域：文件上传（单文件、批量、分片）、文件下载（直接下载、断点续传）、上传下载进度监控、上传下载权限校验，支撑不同大小、不同类型文件的高效传输，适配大文件处理场景，提升用户与运营人员操作体验。

3. 文件处理域：文件格式校验、文件大小限制、文件格式转换（如图片压缩、格式转换）、文件水印（文字水印、图片水印）、文件预览，确保文件合规、适配各业务场景展示需求，同时优化文件体积，提升传输与加载效率。

4. 文件管理域：文件元数据管理、文件分类、文件查询、文件删除、文件归档、文件版本管理，留存所有文件操作记录用于追溯，同时支持用户与运营人员的文件操作与管理，满足合规与业务追溯需求，实现文件全生命周期管理。

5. 安全管控域：文件安全校验（病毒扫描、恶意文件拦截）、访问权限管控、防盗链、文件脱敏、操作日志审计，防范文件泄露、恶意上传等安全风险，确保文件资源安全合规，同时满足监管与审计要求。

# 二、整体架构说明

本文件服务模块基于微服务拆分设计，核心遵循**存储安全、传输高效、数据不跨域、权限可控**原则，坚守微服务数据私有、接口交互的核心准则，适配高并发文件上传下载场景，确保文件传输时效与存储可靠性，同时与其他服务解耦，采用无状态设计，支持水平扩展，提升系统可维护性与扩展性，贴合企业级文件服务的高性能与高可用要求：

- mall\-auth：仅做鉴权，不参与文件存储、上传下载、格式转换等逻辑，仅提供用户、运营人员、商家的身份校验与角色权限支撑，确保文件操作的安全性与权限合规。

- mall\-user：提供用户ID、用户头像、用户权限等信息，用于用户相关文件（头像、个人凭证）的关联存储与权限管控；接收文件模块的用户文件变更回调（如头像更新），同步用户相关文件信息，不修改用户核心数据，仅支撑用户文件场景的协同。

- mall\-product：发起商品相关文件请求（商品图片、商品说明书、规格参数文件），接收文件上传完成回调与文件地址同步，支撑商品详情展示、商品审核等场景；提供商品ID、商品分类等信息，用于商品文件的分类存储与关联管理，确保商品文件与商品信息的一致性。

- mall\-order：发起订单相关文件请求（订单凭证、发票、物流单据），接收文件上传完成回调与文件地址同步，支撑订单审核、售后维权、订单归档等场景；提供订单ID、订单状态等信息，用于订单文件的关联存储与权限管控，确保订单文件的可追溯性。

- mall\-marketing：发起营销相关文件请求（活动海报、营销视频、优惠券图片），接收文件上传完成回调与文件地址同步，支撑营销活动展示、活动审核等场景；提供活动ID、活动类型等信息，用于营销文件的分类存储与管理，助力营销素材的高效复用。

- mall\-aftersale：发起售后相关文件请求（售后凭证、退货照片、维修单据），接收文件上传完成回调与文件地址同步，支撑售后审核、维权处理等场景；提供售后ID、售后类型等信息，用于售后文件的关联存储与权限管控，提升售后处理效率。

- mall\-file：**唯一负责文件存储、上传下载、格式处理、文件管理、安全管控**，独享自身业务数据（文件元数据、存储配置、操作日志、安全记录），不允许其他服务直接访问数据库与存储介质，仅通过接口提供文件上传、下载、查询、删除等服务，支持本地存储、阿里云OSS、腾讯云COS等多存储介质灵活切换，采用无状态设计支持水平扩展，支撑全平台文件服务场景，同时可适配容器化部署，降低运维成本、提升扩展性。

核心边界：

- 只做“文件相关”，不修改其他业务模块数据、不参与业务逻辑处理，仅接收各模块文件请求、完成文件上传下载存储与处理、同步文件状态，专注于文件资源管理核心能力，遵循微服务高内聚低耦合原则。

- 所有文件操作（上传、下载、删除、转换、归档）均有日志记录、可追溯，文件上传下载时效满足业务需求（MVP阶段单文件上传≤3s、下载≤2s，拓展阶段大文件分片上传≤10s/GB、断点续传无缝衔接），适配高并发上传下载场景（如商品批量上新、大促活动素材上传），同时通过异步处理、流式传输提升处理效率，应对峰值流量。

- 文件存储统一管理，支持多存储介质适配与灵活切换，确保文件存储安全可靠，同时支持文件容量监控与扩容预警，避免存储不足影响业务运转，贴合企业级文件存储的高可用需求。

- 支持文件上传下载状态实时回调，操作异常可自动重试，确保文件传输可靠性；同时支持精细化权限管控，不同角色仅可操作对应权限的文件，防范文件安全风险，兼顾安全与便捷性。

# 三、初阶段业务流程（当前落地核心）

核心目标：支撑商城基础文件服务场景，完成核心业务文件的上传、下载、存储、基础查询，覆盖用户端、商家端、运营端基础文件需求，实现文件与各核心业务模块的基础协同，确保文件传输及时、存储安全，采用简单高效的存储与传输方式，降低初期开发成本，同时完成基础存储架构搭建，支撑核心业务落地。

## 3\.1 核心业务流程（7大必备）

1. **存储基础配置（运营后台）**：运营人员经mall\-auth鉴权\+管理员角色校验通过后，配置基础存储策略（默认本地存储），设置文件存储路径、单个文件大小限制（如≤100MB）、支持的文件格式（如图片jpg/png、文档pdf/docx）；配置文件存储容量预警阈值，当存储容量达到阈值时触发提醒，同时配置基础文件分类规则（按业务模块分类：商品、订单、用户、营销），完成文件存储的基础配置，确保存储架构贴合初期业务需求。

2. **文件基础上传（多端支持）**：用户、商家、运营人员经mall\-auth鉴权\+对应角色校验通过后，发起文件上传请求（单文件上传），传入文件、业务关联ID（如商品ID、用户ID）、文件类型；系统校验文件格式、大小合法性，校验通过后将文件存储至指定路径，生成唯一文件ID与访问地址，记录文件元数据（文件名、大小、格式、上传时间、上传人、业务关联ID），完成文件基础上传，同时支持上传进度实时反馈，提升用户操作体验。

3. **文件基础下载（多端支持）**：用户、商家、运营人员经mall\-auth鉴权\+对应权限校验通过后，通过文件ID或访问地址发起文件下载请求；系统校验访问权限（仅允许文件关联业务的相关角色下载），校验通过后返回文件流，支持直接下载，下载完成后记录下载日志（下载人、下载时间、文件ID），确保文件下载的权限合规与可追溯。

4. **文件基础处理（后台自动）**：文件上传完成后，系统自动对文件进行基础处理，图片文件自动压缩（适配展示需求，如商品图片压缩至合适尺寸），所有文件进行格式校验与简单安全扫描（拦截明显恶意文件）；处理完成后更新文件状态（已处理、未处理），处理失败时标记异常，通知上传人重新上传，确保文件适配业务展示需求，同时防范基础安全风险。

5. **文件基础查询与管理（多端支持）**：运营人员可在后台按文件类型、业务关联ID、上传时间、上传人筛选查询文件，查看文件详情（元数据、存储路径、访问地址）；支持单个文件删除（删除文件同时删除存储文件与元数据），删除前校验权限（仅管理员、文件上传人可删除）；用户可在个人中心查询自身上传的文件（如头像、个人凭证），支持头像文件替换，满足用户与运营的基础文件管理需求。

6. **文件上传下载异常基础处理（后台自动\+手动）**：当文件上传失败（如存储路径不存在、文件过大）、下载失败（如文件损坏、权限不足）时，系统自动记录异常日志，标记文件状态为“上传失败”“下载失败”；针对上传失败，自动重试1次（间隔1分钟），重试失败后通知上传人；运营人员可在后台查看异常记录，手动删除异常文件、触发重新上传，确保核心文件的有效存储与访问，同时记录异常原因，为后续优化提供支撑。

7. **文件状态回调（后台自动）**：文件上传、下载、删除完成后（无论成功/失败），系统自动回调发起文件请求的业务模块，同步文件状态（上传成功/失败、下载成功/失败、已删除）与文件访问地址，确保各业务模块及时掌握文件情况，支撑业务流程的闭环协同（如商品图片上传完成后，同步至商品模块更新商品详情），同时记录回调日志，便于问题追溯。

# 四、后期拓展完整版业务流程（企业级升级）

在MVP基础上，补齐多存储介质适配、大文件处理、精细化文件管理、高级安全管控、文件数据分析，形成企业级文件中台，遵循微服务独立扩展原则，提升文件上传下载效率、存储可靠性、安全合规性与用户体验，适配高并发、多场景、精细化运营需求，完善文件全生命周期管理，同时融入智能优化与风控机制，降低运维成本，贴合企业级文件服务的高性能、高可用、高安全需求。

## 4\.1 新增拓展业务流程（10大核心，覆盖五大核心域）

1. **多存储介质精细化管理（文件存储域）**：新增多存储介质适配（本地存储、阿里云OSS、腾讯云COS、华为云OBS等），支持按业务场景配置存储策略（如商品图片存储至阿里云OSS，敏感文件存储至本地加密存储）；新增存储介质切换功能（支持无缝切换，不影响已存储文件访问），支持存储分片、存储备份与恢复机制，避免文件丢失；新增存储容量精细化监控、扩容提醒与自动扩容配置，适配海量文件（千万级以上）存储场景，同时支持存储性能监控，优化存储资源分配，贴合企业级文件存储的高可用与可扩展需求。

2. **大文件与批量文件处理（文件上传下载域）**：新增大文件分片上传/断点续传功能，支持GB级大文件上传，断点续传可保留上传进度，避免重复上传，同时支持HTTP Range请求，提升大文件下载效率；新增批量文件上传/下载功能，支持多文件同时上传下载，可配置上传下载并发数，提升批量操作效率；新增上传下载限流控制，适配高并发场景，同时引入异步处理机制，对于耗时较长的大文件上传下载任务，采用消息队列异步处理，不阻塞前端操作，提升用户体验。

3. **文件高级处理（文件处理域）**：新增文件格式转换功能（如图片格式互转、文档转PDF、视频格式压缩），支持批量转换；新增文件水印精细化配置（文字水印、图片水印，可配置水印位置、透明度、大小），支持按业务场景自动添加水印（如商品图片添加店铺水印、凭证文件添加平台水印）；新增文件预览功能（支持多种格式文件在线预览，如pdf、docx、图片、视频），无需下载即可查看文件内容；新增图片智能裁剪、缩放，适配不同展示场景（如APP端、PC端商品图片尺寸适配），提升文件展示效果，同时优化文件体积，降低存储与传输成本。

4. **文件精细化管理（文件管理域）**：新增文件归档功能，对超过一定时间（如6个月）的非活跃文件自动归档（可配置归档存储介质），支持归档文件查询、恢复与彻底删除；新增文件版本管理，文件修改后保留历史版本，可回滚至任意历史版本，支撑文件修改追溯；新增文件标签管理，可给文件添加标签，便于快速检索；支持文件全文检索，可通过关键词、文件名、上传人、业务关联ID快速检索文件；新增多端文件同步（H5、小程序、PC端），确保不同端文件操作一致，同时完善文件元数据管理，记录文件所有操作轨迹，满足合规与追溯需求。

5. **高级安全管控（安全管控域）**：新增文件安全高级校验，对接第三方病毒扫描接口，全面拦截恶意文件、病毒文件，支持可疑文件隔离与手动审核；新增文件访问权限精细化配置（如按角色、按业务范围配置文件访问权限，支持临时授权访问）；新增防盗链配置（支持Referer白名单、URL签名防盗链），防止文件被非法盗用；新增文件脱敏处理（如用户凭证文件隐藏敏感信息），保护用户隐私；完善操作日志审计，全量记录所有文件操作（上传、下载、删除、修改），支持日志检索、导出，满足合规审计要求，同时搭建安全预警机制，针对恶意上传、非法访问等行为触发预警，通知运维人员及时处理。

6. **文件权限精细化管控（全域联动）**：新增角色权限精细化配置，区分管理员、运营人员、商家、普通用户的文件操作权限（如商家仅可操作自身店铺相关文件，普通用户仅可操作个人文件）；支持文件共享功能，可将文件共享给指定角色或用户，配置共享有效期；新增文件访问日志精细化统计，记录每一次文件访问的详细信息（访问人、访问时间、访问IP、访问方式），便于安全追溯；支持文件操作审批流程（如敏感文件删除、批量文件上传需管理员审批），确保文件操作合规，防范安全风险。

7. **文件协同与复用（全域联动）**：新增文件共享池，运营人员可将常用营销素材、模板文件存入共享池，供各业务模块、商家快速复用，提升工作效率；新增文件协同编辑功能（支持多用户同时编辑文档类文件），实时同步编辑内容，支撑团队协同工作；新增文件关联管理，建立文件与业务数据的关联关系（如商品图片与商品SKU关联、订单凭证与订单关联），删除业务数据时可选择同步删除关联文件或保留归档，提升文件管理的便捷性，同时避免文件冗余。

8. **文件数据精细化统计与分析（文件管理域）**：搭建文件数据看板，实时展示核心数据（文件上传量、下载量、存储容量占用、文件类型占比、上传下载成功率、异常率、各存储介质使用率）；支持按时间、业务模块、用户角色、文件类型筛选数据；生成文件分析报告，分析文件存储瓶颈（如某存储介质占用过高、某类型文件上传频繁）、文件操作行为（如用户常用文件类型、高频操作时段），为存储策略优化、权限调整、资源分配提供数据支撑；支持自定义报表导出（PDF/Excel格式），满足不同运营与运维需求，助力精细化运营决策。
