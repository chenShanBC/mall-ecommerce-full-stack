# 三、完整登录流程（基于OAuth2\.0标准）

1. **前端触发登录**：用户点击页面中的支付宝登录图标，前端请求后端接口，获取支付宝授权跳转地址。

2. **跳转支付宝授权页**：后端拼接支付宝官方授权URL，前端跳转至该页面，用户确认授权。

3. **授权回调**：用户同意授权后，支付宝自动回调至配置的「redirect\-uri」，并携带参数auth\_code（授权码）和state（防篡改随机值）。

4. **换取访问令牌**：后端接收auth\_code，调用支付宝接口「alipay\.system\.oauth\.token」，换取access\_token（访问令牌）。

5. **获取用户信息**：使用access\_token调用支付宝接口「alipay\.user\.info\.share」，获取用户核心信息（支付宝唯一ID、昵称、头像）。

6. **本地账号逻辑处理**：
        

    - 已绑定：根据支付宝唯一ID查询第三方绑定表，找到关联的本地用户ID，直接签发JWT登录令牌。

    - 未绑定：自动创建本地普通用户（手机号留空，昵称、头像同步支付宝信息），并在第三方绑定表中建立关联关系，再签发JWT令牌。

7. **登录完成**：后端重定向前端首页，并携带JWT令牌，前端存储令牌，维持用户登录状态。

# 五、后端核心接口设计

## 5\.1 生成支付宝授权跳转地址接口

- **接口地址**：/api/third/alipay/auth

- **接口作用**：拼接支付宝官方授权URL，携带回调地址、防篡改state参数，返回给前端用于跳转。

- **授权URL模板**（需对redirect\_uri进行URL编码）：

```plain text
https://openauth.alipay.com/oauth2/publicAppAuthorize.htm?
app_id=你的APPID
&scope=auth_user
&redirect_uri=URL编码后的回调地址
&state=随机唯一值（用于防跨站攻击）
```

补充：state参数建议使用UUID生成，后端需存储该值，回调时校验一致性，防止CSRF攻击。

## 5\.2 支付宝授权回调接口

- **接口地址**：/api/third/alipay/callback（与开放平台配置的回调地址一致）

- **接收参数**：auth\_code（授权码）、state（随机值）

- **核心业务逻辑**：
        

    1. 校验state参数：对比后端存储的state值，不一致则拒绝处理，防止跨站攻击。

    2. 换取access\_token：调用支付宝「alipay\.system\.oauth\.token」接口，传入app\_id、private\_key、auth\_code等参数，获取access\_token和支付宝用户ID。

    3. 获取用户信息：调用支付宝「alipay\.user\.info\.share」接口，传入access\_token，获取用户昵称、头像等信息。

    4. 账号绑定/登录：查询user\_third\_bind表，根据third\_type=3和third\_uid（支付宝用户ID）判断是否已绑定，执行对应逻辑（已绑定则登录，未绑定则自动注册并绑定）。

    5. 返回结果：生成JWT令牌，重定向前端首页，并将令牌携带在URL中（或通过cookie、localStorage存储）。

# 七、业务规则设计

1. **自动注册规则**：支付宝登录时，若未查询到绑定的本地账号，自动创建新用户，默认昵称、头像同步支付宝信息，手机号留空，密码可设为随机值（无需用户输入）。

2. **账号互通规则**：
        

    - 手机号注册的用户，可在个人中心手动绑定支付宝账号，绑定后下次可通过支付宝一键登录。

    - 支付宝登录创建的用户，可后续在个人中心绑定手机号，完善账号信息（方便找回密码、接收订单通知）。

3. **登录状态规则**：支付宝登录与手机号登录共用一套JWT令牌、拦截器、会话体系，确保登录状态统一，退出登录时仅清空本地令牌，不解除第三方绑定关系。

4. **异常处理规则**：
        

    - 授权码过期：重新引导用户点击支付宝登录，获取新的auth\_code。

    - 签名异常：检查公私钥配置是否正确，确保与开放平台一致。

    - 用户信息获取失败：提示用户重新授权，或降级为仅完成登录（不同步昵称头像）。
