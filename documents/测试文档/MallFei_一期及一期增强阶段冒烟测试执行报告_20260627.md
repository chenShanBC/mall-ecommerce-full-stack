## 七、主要跳过项归类

| 跳过类别 | 典型表现 | 原因分析 | 影响评估 |
|---|---|---|---|
| C端登录态不足 | 手机号绑定状态、支付状态同步、查询支付单等返回未登录或登录失效 | 用户密码登录需要验证码参数，未取得 C 端用户 token | 影响 C 端增强接口完整自动化验证，但不影响后台链路验证 |
| 参数校验不满足 | 收货人、异常类型、审核订单号、调整原因、业务类型、调整数量等不能为空 | 部分接口需要更完整的业务请求体或有效业务上下文 | 属于测试数据和请求构造问题，建议后续补齐专项用例 |
| 支付配置缺失 | 支付宝授权地址、支付回调返回支付配置不完整或支付渠道为空 | 本地环境未配置完整第三方支付参数 | 不影响 Mock 或后台支付查询能力，影响真实支付通道验证 |
| 业务状态不满足 | 支付异常修复、订单发货、确认收货、售后归档等提示当前状态不允许操作 | 当前订单、支付单、对账单或售后单已处于非目标状态 | 非 Strict 模式下作为环境状态差异处理，不计入失败 |
| 有效业务 ID 不足 | 售后操作、售后跟踪、售后转人工、售后收货等需要有效 ID | 当前列表未返回可继续流转的目标记录 | 需要准备专门的售后测试数据后再验证 |
